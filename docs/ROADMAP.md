# Roadmap

This document outlines planned improvements for the `OMCHOKSI108/VibeThinkerModel` fork. The focus is on documentation, tooling, and accessibility — not on modifying the original model.

## Short Term

- [x] Fork original repository with clear naming
- [x] Preserve original README, license, and attribution
- [x] Create `docs/` with structured documentation
- [x] Mirror model weights to Hugging Face with improved model card
- [x] Add repository topics for discoverability

## Medium Term

- [ ] Add Colab / Jupyter inference notebook
- [ ] Add example scripts for common use cases
- [ ] Add Dockerfile for reproducible inference environment
- [ ] Add quantization guides (bitsandbytes, AWQ, GPTQ)
- [ ] Add FAQ based on community questions
- [ ] Add comparison table of VibeThinker variants

## Long Term

- [ ] Community-contributed examples (if any)
- [ ] Integration guides for common frameworks (LangChain, LlamaIndex)
- [ ] Performance benchmarks on standard hardware
- [ ] Model weight mirrors in additional formats (GGUF, ONNX) — if permissive

## Non-Goals

- Modifying original model weights
- Fine-tuning or creating derivative models
- Claiming ownership or authorship
- Adding features not aligned with the original project scope
- Replicating or competing with the original project

## Contributions

Contributions that improve documentation, fix bugs in the original eval code, or add tooling are welcome. Please open an issue first to discuss.
