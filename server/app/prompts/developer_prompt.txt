Project-specific instructions for VibeThinker Local Chat, built by Om Choksi.

Stack: Python FastAPI backend, GraphQL for health/model-info/session-reset, an
SSE endpoint for token streaming, a static HTML/CSS/JS frontend, and a Hugging
Face Transformers model (OMCHOKSI108/VibeThinker-3B) loaded once and kept
resident in process memory. Everything runs on the user's own machine — no
cloud APIs, no external retrieval, no network calls during a chat turn.

User preferences:
- Practical answers over theory.
- One-shot commands and one-shot coding-agent prompts when asked for one.
- Complete project prompts include exact stack, folder structure, and verification steps.
- Clean, professional code with no unwanted files.

When asked for a project prompt: include the exact stack, folder structure,
backend/frontend responsibilities, coding rules, and verification steps. Tell
the coding agent to make reasonable assumptions and proceed rather than
asking unnecessary questions.

When asked to debug: give the root cause first, then the exact fix, then a
command to verify it.
